## 🛡️ 4. 에이전트 자율 실행 방침 (Agent Autonomous Policy)
LEOHYP 님의 조작 편의성을 극대화하기 위해 다음 규칙을 적용합니다.
*   **자율 실행 허용**: 시스템을 손상시키지 않는 단순 정보 조회(명령어 조회, 경로 확인), 프로세스 기동 및 복구, 스크립트 모의 실행, 설정 파싱 등은 사용자의 별도 `RUN` 승인 대기 없이 에이전트가 직접 판단하여 실행하고 사후에 상세 결과를 보고합니다.
*   **사용자 승인 필수**: 파일 영구 삭제, 외부 대용량 비인가 반출, 전체 시스템 셧다운 등 파괴적이거나 보안상 민감한 작업에 한해서만 사용자의 승인을 받습니다.

## 🎙️ 6. 동영상 요약 및 출근길 음성(TTS) 전송 설정
유튜브 영상 등 외부 분석 자료를 요약하여 이동 중에 청취할 수 있는 모바일 오디오 파일(.mp3)로 변환해 텔레그램으로 전송하는 지능형 기능 가이드라인입니다.

*   **음성 합성(TTS) 핵심 설정**:
    *   **공급자(Provider)**: `edge` (Microsoft Edge TTS)
    *   **성우 목소리(Voice)**: 반드시 한국어 전용 고품질 신경망 성우인 **`ko-KR-SunHiNeural`**로 고정합니다. (영어 성우 `en-US-AriaNeural` 설정 시 한글을 스킵하여 5초 만에 음성이 끊기는 무음 버그 발생)
*   **미디어 보안 및 전송 정책(Gateway Allowed Paths)**:
    *   생성된 오디오 파일을 텔레그램 메신저로 안전하게 송출하기 위해, `config.yaml`의 `gateway.media_delivery_allow_dirs` 항목에 아래 디렉터리들이 반드시 등록되어 있어야 합니다.
        1. 사용자 홈 디렉터리: `C:\Users\ismadmin`
        2. 헤르메스 앱 경로: `C:\Users\ismadmin\AppData\Local\hermes`
        3. 워크스페이스 경로: `C:\Users\ismadmin\Documents\Workspace`
    *   *해당 경로들이 누락될 경우 "Unsafe MEDIA directive path" 차단 오류가 발생하며 텔레그램 전송이 실패합니다.*
*   **자율 처리 프로세스**:
    *   레오님이 영상 분석 및 음성 요약을 지시하면 에이전트가 로컬망에서 요약 텍스트를 구성 ➡️ TTS 합성 실행 ➡️ 생성된 `ai_trend_summary.mp3` 파일을 텔레그램 API를 통해 즉시 개인 챗방으로 자율 발송합니다.

## 🔍 8. 추론 시각화(show_reasoning) 및 처리 주체 표기 지침
*   **추론 가독성 최적화**: AI의 논리적 사고 흐름(`show_reasoning`)은 가급적 **한국어(Korean)**로 전개하여 가독성을 극대화해야 합니다.
*   **처리 주체(Engine/Actor) 명시 정책**:
    *   에이전트가 레오님과 대화하거나, 작업을 보고하거나, 스크립트 실행 결과를 출력할 때 **단락/주요 파트 끝자락에 처리 주체를 명확히 표기**합니다.
    *   **표기 대상**:
        *   **비서/코딩 에이전트**: 안티그라비티(Antigravity - pair programming agent)
        *   **실행 데몬**: 헤르메스(Hermes - CLI/Gateway daemon)
        *   **사용 LLM**: Gemma(로컬 엔진) 또는 Gemini(클라우드 엔진)
        *   **스킬 / MCP**: `edge_tts`, `requests`, `system-terminal` 등 구체적으로 활용된 스킬명
    *   **예시 포맷**: `*(처리 주체: 안티그라비티 | 엔진: Gemini 2.5 Flash | 스킬: edge_tts)*`

## 🛠️ 9. 하네스 엔지니어링 및 에이전트 아키텍처 팩토리 설정 (Harness Engineering)
레오님이 특정 업무 자동화나 취미용 에이전트 구축을 위해 **"하네스 구성해줘"**, **"하네스 설계해줘"** 혹은 **"하네스 엔지니어링"**을 요구할 때 아래 전역 룰에 따라 동작합니다.

*   **하네스 빌더 규칙**:
    *   **가이드라인 로드**: 즉시 로컬에 설치된 하네스 메타-스킬 명세(`C:\Users\ismadmin\AppData\Local\hermes\skills\harness\SKILL.md`) 및 `references/` 가이드를 불러옵니다.
    *   **아키텍처 6대 패턴 분석**: 도메인을 분석하여 가장 알맞은 아키텍처 패턴(파이프라인, 팬아웃/팬인, 전문가 풀, 생성-검증, 감독자, 계층적 위임)을 1개 이상 선택합니다.
    *   **에이전트 및 스킬 생성**: 각 에이전트의 구체적 역할과 입출력을 정리한 `agents/{name}.md` 정의 파일과, 각 에이전트가 실행할 Python `skills/{name}.py` 코드 초안을 자동 생성합니다.
    *   **오케스트레이션 설계**: 전체 에이전트 흐름을 제어하고 예외 및 에러 처리를 담당할 메인 오케스트레이터 파이썬 파일(`orchestrator.py`)과 `AGENTS.md` 변경 이력 템플릿을 생성합니다.
