# 🤖 Harness Engineering 운영 가이드 & 카탈로그 (Hermes-Antigravity Edition)

본 가이드는 헤르메스(Hermes)와 안티그라비티(Antigravity) 생태계에 이식된 **하네스 메타-스킬(Harness Meta-Skill)**을 통해 구축할 수 있는 에이전트(Agents) 및 스킬(Skills)의 카탈로그와 구체적인 사용 방법을 정의합니다.

---

## 🧭 1. 하네스 6대 아키텍처 패턴 카탈로그

업무 자동화 및 개인 취미용 하네스 팀을 구성할 때 선택할 수 있는 6가지 검증된 구조적 패턴입니다.

| 패턴명 (Pattern) | 구조 특징 | 추천 개인/RPA 도메인 |
| :--- | :--- | :--- |
| **① 파이프라인 (Pipeline)** | 순차적 흐름 (A ➔ B ➔ C)<br>이전 결과물이 다음의 입력이 됨 | 일일 백업 로그 분석 ➔ 기안문 작성 ➔ 결재 상신 자동화 |
| **② 팬아웃/팬인 (Fan-out/In)** | 독립적인 병렬 조사 후 결과 취합 | 뉴스 및 IT 트렌드 소스 동시 병렬 크롤링 ➔ 보고서 합성 |
| **③ 전문가 풀 (Expert Pool)** | 조건에 맞는 특정 에이전트만 호출 | 서버 오류 유형 분석 ➔ (DB/웹서버/네트워크) 전문가 선별 할당 |
| **④ 생성-검증 (Producer-Reviewer)** | 생성자와 검증자가 쌍으로 자율 피드백 | 맛집 마크다운 일기 생성 ➔ 주소/영업 여부 지도 API 실시간 검증 |
| **⑤ 감독자 (Supervisor)** | 리더가 작업 현황을 감시하며 동적 배분 | 수백 장의 사진 EXIF GPS 추출 작업을 여러 워커에 분산 처리 |
| **⑥ 계층적 위임 (Hierarchical)** | 상위 에이전트가 하위에 순차 위임 | 대규모 프로젝트 기획 ➔ 프론트엔드/백엔드 하브 에이전트 위임 |

## 👥 2. 사용 가능한 하네스 에이전트(Agents) 카탈로그

하네스 빌더가 자동으로 구성하고 레오님이 바로 활용할 수 있는 대표적인 전담 전문가 에이전트 명세입니다.

### 1. `Gourmet-Finder (Gemma/Local)`
*   **핵심 역할**: 이미지 파일 내 EXIF 데이터 파싱 및 위치 정보 추출.
*   **입력**: 레오님이 송신한 원본 맛집/여행 사진(`.jpg`, `.png`).
*   **출력**: GPS 위경도 좌표 및 촬영 날짜 메타데이터.

### 2. `Fact-Checker & QA (Gemini/Cloud)`
*   **핵심 역할**: 수집되거나 생성된 결과물의 무결성 검증.
*   **입력**: 초안 보고서 또는 타깃 데이터.
*   **출력**: 검증 결과 리포트 (`PASS` / `REDO` 피드백 루프).
*   **검증 방식**: 지도 API 실시간 핑, CVE 취약점 공식 DB 대조 등.

### 3. `Doc-Builder (Gemini/Cloud)`
*   **핵심 역할**: 검증된 데이터를 Obsidian 형식의 문서로 릴리즈.
*   **입력**: 최종 합의된 텍스트 및 메타데이터.
*   **출력**: YAML Frontmatter가 포함된 마크다운 파일 (`.md`).

### 4. `Audio-Director & DJ (Gemini/Cloud)`
*   **핵심 역할**: 마크다운 문서를 자연스럽게 귀로 듣는 낭독용 1인 라디오 대본으로 가공.
*   **입력**: 정갈한 보고서 본문.
*   **출력**: 대화체 낭독 스크립트.

## 🛠️ 3. 사용 가능한 표준 하네스 스킬(Skills) 카탈로그

에이전트들이 도구 상자에서 꺼내어 실제로 행동을 실행하는 재사용 가능한 핵심 라이브러리 목록입니다.

1.  **`edge_tts` (음성 합성)**:
    *   한국어 최고 품질 신경망 성우(`ko-KR-SunHiNeural`)를 기동하여 텍스트를 고음질 오디오 파일(`.mp3`)로 굽는 스킬.
2.  **`pydub_bgm_mixer` (BGM 오버레이)**:
    *   오디오 파일 뒷단에 레오님이 지정한 배경 음악(BGM)을 볼륨 조절하여 자연스럽게 배경음으로 믹싱해주는 스킬.
3.  **`telegram_publisher` (채널 배송)**:
    *   완성된 보고서 텍스트 및 `.mp3` 오디오 파일을 텔레그램 개인 챗방으로 즉시 송출하는 스킬.
4.  **`gps_exif_parser` (메타데이터 추출)**:
    *   사진 원본 바이너리를 분석해 위도, 경도, 카메라 기종, 촬영 시간 등의 GPS 메타데이터를 Dictionary 형태로 리턴하는 스킬.
5.  **`obsidian_sync` (보관소 연동)**:
    *   생성된 일지나 분석 리포트를 지정된 로컬 Obsidian Vault 경로에 자동 생성/인덱싱해주는 스킬.

## 🚀 4. 하네스 메타-스킬 사용 방법 (Usage Guide)

### 1단계: 하네스 설계 및 생성 요청 (안티그라비티 트리거)
레오님이 대화창에 다음과 같이 자연어로 요청합니다:
> 💬 *"레오의 출근길 IT 팟캐스트 제작을 위한 **하네스 설계해줘**"*
> 💬 *"사진첩 정리 자동화를 위한 **하네스 구성해줘**"*

안티그라비티는 요청을 감지하면 즉시 로컬에 튜닝된 `skills/harness/SKILL.md`를 로드하여 다음을 자율 수행합니다:
1.  도메인을 분석하고 6대 패턴 중 최적의 협업 모델을 매칭합니다.
2.  `Workspace/agents/` 디렉토리에 필요한 에이전트 정의 마크다운 파일(`.md`)들을 자동 생성합니다.
3.  `Workspace/skills/` 디렉토리에 각 에이전트가 호출할 파이썬 실행 모듈(`.py`)들을 작성합니다.
4.  전체 흐름을 제어하고 에러 발생 시 피드백 루프를 처리할 메인 오케스트레이터 파일(`orchestrator.py`)을 자동 빌드합니다.

### 2단계: 프로젝트 결합 및 로드 (SSOT 포인터 등록)
생성이 완료되면 안티그라비티가 워크스페이스의 마스터 콘텍스트인 `AGENTS.md` 맨 아래에 다음과 같은 하네스 변경 이력을 고정합니다:
```markdown
## 🤖 하네스: Gourmet-Harness
* **목표**: 맛집 사진 GPS 파싱 및 지도 시각화
* **트리거**: 텔레그램 맛집 사진 수신 시 `gourmet_orchestrator.py` 실행
* **변경 이력**: YYYY-MM-DD - 초기 구성 완료
```

### 3단계: 하네스 팀 자율 실행 및 모니터링
1.  **트리거**: 레오님이 텔레그램으로 사진을 보내거나, 윈도우 스케줄러 정기 시각이 되면 `python orchestrator.py`가 백그라운드에서 실행됩니다.
2.  **자율 협업**: 에이전트들이 `_workspace/` 폴더를 거점으로 데이터를 주고받으며 검수 및 빌드 작업을 무인으로 완료합니다.
3.  **결과 확인**: 최종 완성본 마크다운이 레오님의 Obsidian 보관소에 들어오고, 동시에 텔레그램으로 낭독 음성 캐스트 알림이 전송됩니다.
