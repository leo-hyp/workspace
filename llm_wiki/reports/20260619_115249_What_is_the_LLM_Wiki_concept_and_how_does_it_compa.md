# LLM 위키 개념 및 기존 RAG와의 비교 보고서

## 서론

이 보고서는 [[Andrej Karpathy]]가 2026년 4월에 제안한 '[[LLM Wiki]]' 개념을 설명하고, 이것이 기존의 '[[Retrieval Augmented Generation (RAG)]]' 시스템과 어떻게 비교되는지 분석합니다. LLM 위키는 대규모 언어 모델(LLM)의 지식 유지 및 합성 문제를 해결하기 위한 새로운 접근 방식을 제시합니다.

## LLM 위키 개념

[[LLM Wiki]]는 [[Andrej Karpathy]]가 2026년 4월에 제안한 개념입니다. 이는 전통적인 위키 및 지식 기반을 "LLM 프로그래머"를 위한 "코드베이스"로 재구상합니다. 이 개념에서 [[Obsidian]]은 LLM이 위키 "코드베이스"와 상호작용하고 관리하는 "IDE"(통합 개발 환경) 역할을 합니다.

LLM 위키의 핵심 목표는 기존 [[Retrieval Augmented Generation (RAG)]] 시스템의 근본적인 문제, 즉 지식 유지 및 합성의 어려움으로 인해 정보가 "잊혀지는" 문제를 해결하는 것입니다. 지식을 상호 연결된 위키 형태로 구조화함으로써, LLM은 정보를 더욱 효과적으로 구축하고 합성할 수 있습니다.

## 기존 RAG (검색 증강 생성)

[[Retrieval Augmented Generation (RAG)]]은 대규모 언어 모델(LLM)의 응답 생성 능력을 향상시키기 위한 아키텍처 패턴입니다. RAG는 LLM이 응답을 생성하기 전에 먼저 지식 기반에서 관련 정보를 검색함으로써 작동합니다. 이러한 방식은 LLM의 응답이 사실적인 데이터에 기반하도록 돕고, 환각(hallucination) 현상을 줄이는 데 기여합니다.

## LLM 위키와 기존 RAG의 비교

[[Andrej Karpathy]]의 [[LLM Wiki]] 개념은 전통적인 [[Retrieval Augmented Generation (RAG)]] 시스템의 특정 한계를 해결하는 것을 목표로 합니다.

| 특징          | LLM 위키                                         | 기존 RAG                                             |
| :------------ | :----------------------------------------------- | :--------------------------------------------------- |
| **목표**      | 더 강력한 지식 유지 및 합성                      | 사실적 데이터에 기반한 응답 생성 및 환각 감소        |
| **지식 구조** | 상호 연결된 위키 (LLM의 "코드베이스" 역할)       | 지식 기반 (정보 검색의 원천)                         |
| **주요 장점** | 정보의 효율적인 구축 및 합성, "잊혀지는" 문제 해결 | 관련 정보 검색을 통한 응답의 근거 마련               |
| **해결 과제** | 전통적인 RAG의 지식 유지 및 합성 부족 문제       | -                                                    |
| **비유**      | LLM은 "프로그래머", 위키는 "코드베이스", Obsidian은 "IDE" | LLM이 지식 기반에서 "검색"하여 "생성"하는 패턴      |

기존 RAG 시스템은 지식 유지 및 합성 측면에서 어려움을 겪으며, 이로 인해 정보가 "잊혀지는" 현상이 발생할 수 있습니다. LLM 위키는 지식을 마치 LLM이 다루는 코드베이스처럼 구조화함으로써 이러한 RAG의 한계를 직접적으로 개선하고자 합니다. 즉, LLM 위키는 단순히 정보를 검색하여 활용하는 것을 넘어, LLM이 지식을 적극적으로 구축하고, 연결하며, 시간이 지남에 따라 합성하여 더욱 견고한 지식 기반을 형성할 수 있도록 돕는 것을 목표로 합니다.

## 결론

LLM 위키는 기존 RAG 시스템의 지식 유지 및 합성 문제를 해결하기 위해 고안된 혁신적인 개념입니다. [[Andrej Karpathy]]가 제안한 이 접근 방식은 LLM이 지식을 "코드베이스"처럼 다루고, [[Obsidian]]과 같은 도구를 "IDE"로 활용하여 정보가 더욱 효과적으로 통합되고 발전할 수 있도록 합니다. 이를 통해 LLM 위키는 전통적인 RAG가 겪는 "정보 망각" 문제를 극복하고, LLM의 지능적인 능력 발전에 새로운 지평을 열 것으로 기대됩니다.
